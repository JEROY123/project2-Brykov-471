# Прогноз опадів за допомогою Machine Learning 🌦

Цей проєкт — це мінісервіс прогнозування погодних умов (а саме наявності опадів) на основі алгоритмів машинного навчання. Застосунок реалізований за допомогою фреймворку Streamlit та дозволяє завантажувати історичні метеодані, навчати модель та отримувати прогноз натисканням кількох кнопок.

## Які дані використовуються

Дані отримуються через HTTP-запити з відкритого безкоштовного сервісу **Open-Meteo** (Archive API). 
Ми використовуємо щоденні (daily) історичні дані за визначений користувачем період (рекомендовано від 1 року для якісного навчання).

**Ознаки (Features), які використовуються для навчання:**
* `temperature_2m_max` — максимальна температура за день;
* `temperature_2m_min` — мінімальна температура за день;
* `wind_speed_10m_max` — максимальна швидкість вітру;
* `precipitation_sum` — сума опадів за день;
* `rain_sum` — сума дощу за день.

**Цільова змінна (Target):**
Цільовою змінною є наявність опадів наступного дня (бінарна класифікація: 0 — опадів немає, 1 — опади є). Вона формується на основі поля `precipitation_sum`, зсунутого на один день назад за допомогою методу `shift(-1)`.

## Яку модель використано

У проєкті використовується алгоритм **Random Forest Classifier (Випадковий ліс)** з бібліотеки `scikit-learn`. 
Цей алгоритм обрано через його високу стійкість до перенавчання (overfitting) та здатність добре працювати з нелінійними залежностями в погодних даних. Модель навчається на 80% завантажених даних, а інші 20% використовуються для тестування та розрахунку метрик (Accuracy, Precision, Recall, F1-Score).

## Структура та організація проєкту

### Файли проєкту:
* `app.py` — головний файл проєкту, який містить логіку інтерфейсу Streamlit, отримання даних з API, підготовку датасету та навчання моделі.
* `weather_daily.csv` — автоматично згенерований файл, у який зберігаються сирі історичні дані після їх завантаження з Open-Meteo.
* `requirements.txt` — список бібліотек Python, необхідних для запуску проєкту.
* `README.md` — опис проєкту, який ви зараз читаєте.

### Послідовність дій у застосунку:
1. **Ініціалізація та Завантаження:** Користувач вводить координати (широту та довготу) і обирає діапазон дат в інтерфейсі. Застосунок робить запит до API Open-Meteo, отримує дані у форматі JSON, перетворює їх у `pandas DataFrame` та виводить на екран таблицю (і паралельно зберігає у CSV).
2. **Підготовка та Навчання:** Після натискання кнопки "Навчити модель", застосунок зчитує дані, формує цільову змінну на "завтрашній" день, відкидає останній рядок (бо для нього ще немає факту "завтра"), розбиває дані на тренувальну і тестову вибірки та тренує `RandomForestClassifier`.
3. **Оцінка:** Після навчання на екран виводяться метрики якості моделі на основі тестової вибірки.
4. **Прогнозування:** Після натискання "Зробити прогноз", застосунок бере дані за *останній отриманий день* (той самий відкинутий рядок) і передає їх у навчену модель, щоб передбачити наявність опадів на наступний день та розрахувати ймовірність цього класу.
